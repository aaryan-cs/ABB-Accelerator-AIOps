Why normalization matters
Raw Prometheus scrape text fed directly to an LLM will hallucinate or crash — L2 normalization is a hard architectural requirement, not an optimization.
The aggregation engine polls Prometheus and Loki every 5 seconds, applies threshold rules (CPU >80%, PVC >90%), and emits clean JSON events. This structured 
output is what makes downstream AI correlation reliable.

Why multi-agent decomposition matters

Single monolithic LLM pipelines fail under high-volume telemetry streams. The multi-agent architecture isolates resource analysis, log intelligence, network tracing, and causal reasoning into specialized inference layers. 
This reduces context overload, improves edge inference speed, and enables reliable root-cause ranking in air-gapped industrial environments.

The entire inference pipeline runs locally using Ollama with Llama 3 8B or Phi-3. No cloud APIs or external services are used at any stage of processing. Ensures complete offline operation for secure and air-gapped 
environments. Reduces latency while maintaining data privacy and deployment independence.

11 Risk Analysis
01 Correlation False Positives

Risk: High Pearson r values between unrelated pods sharing periodic background activity.

✓ Mitigation: Require high r across multiple independent lag windows + eBPF network confirmation before inferring strong 
dependencies.

02 LLM Hallucination on Edge Hardware
Risk: Quantized local models (Phi-3) can produce incorrect summaries given malformed input.

✓ Mitigation: L2 normalization ensures the model only receives clean structured JSON — never raw Prometheus text.

03 eBPF Kernel Compatibility
Risk: eBPF requires specific kernel versions; may be unavailable on some edge K3s distributions.

✓ Mitigation: Graceful degradation to network-policy-based topology inference with reduced edge confidence scores.

04 Single-Node Resource Contention
Risk: Running Prometheus, Loki, eBPF, Go aggregator, and LLM on the same node as workloads.

✓ Mitigation: All observability components in dedicated low-priority namespace with strict CPU/MEM limits; 15s scrape + 5s 
polling tuned for minimum overhead.
